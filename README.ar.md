## ما هو؟

**Code Crime Scene** أداة سطح مكتب محلية (ملف تنفيذي واحد) للمطوّرين والفرق اللي يبون قراءة صادقة وسريعة لصحّة قاعدة الكود — **بدون رفع أي سطر مصدري للسحابة**.

توجّه الأداة لمجلّد، فيمشي محرّك Rust على الشجرة، يشغّل التحليل الساكن **بالكامل على جهازك**، وينتج **تقريرًا بأسلوب "Forensic Noir"**: خمس درجات، وحكم مُقدَّر، وقائمة **ملاحظات قابلة للتنفيذ** — كل ملاحظة بدليلها (ملف + سطر + مقتطف سطر واحد)، ومبرّر خطورة رقمي، وخطة إعادة هيكلة آمنة على شكل PRs.

وفيه رأي ثانٍ **اختياري**: تُرسل خلاصة مضغوطة **بلا أي كود** إلى **DeepSeek** فتحصل على "تقرير المحقّق" مكتوبًا. كودك المصدري ما يطلع من جهازك أبدًا — يُرسل فقط أرقام مجمّعة ومسارات ملفات، وفقط لمّا تطلب أنت.

> **محلّي افتراضيًا. الذكاء بطلبك.** الفحص محلّي 100%، و DeepSeek ما يُستدعى إلا لمّا تضغط **«حلّل بالذكاء»**.

## أبرز المزايا

| | |
|---|---|
| 🔒 **يعمل دون اتصال** | لا backend، لا تسجيل دخول، لا تتبّع. التحليل الساكن يجري محليًا في Rust. |
| 🧠 **ذكاء اختياري** | DeepSeek يعطيك قراءة أعمق مكتوبة — بطلبك، وبخلاصة بلا كود. |
| 🎯 **أدلّة مرتّبة** | الاستخدام الفعلي يتفوّق على الاستيراد/النوع/التعليق. كل ملاحظة بملف وسطر ومقتطف. |
| 📊 **خمس درجات + حكم** | تقييم المشروع، الدَّين التقني، المعمارية، الخطر الأمني، قابلية الصيانة. |
| 🧩 **واعٍ بالسياق** | يميّز الهوك عن نقطة دخول الخادم عن ملف الأيقونات، ويقترح إصلاحًا يناسب الملف. |
| 🩹 **خطط إعادة هيكلة آمنة** | كل ملاحظة معها تقطيع PR-by-PR + أوامر التحقّق (`npm run`) بالضبط. |
| 🌍 **ثنائي اللغة** | واجهة كاملة إنجليزي/عربي (RTL)، والذكاء يردّ بلغتك. |
| 🔑 **المفتاح في خزنة النظام** | مفتاح الـ API في Windows Credential Manager — لا في ملف، ولا في الواجهة. |
| 🗄️ **سجلّ محلّي** | كل تقرير يُحفظ في قاعدة SQLite محلية تفتحها وقت ما تبي. |

## 🔁 جرّبناها على نفسها — حلّينا الأداة بالأداة

أغلب أدوات الجودة ما تُوجَّه أبدًا على كودها هي. نحنا وجّهنا Code Crime Scene على **كوده نفسه** — فأدانَنا.

أول فحص ذاتي كان قاسيًا: **51 / 100 · تقدير F · «جريمة قتل».** وكان **محقًّا**. سمّى ديونًا حقيقية:

- **12.1% كود مكرّر** في المشروع
- **ملفات عملاقة** تتجاوز 400 سطر بكثير — محرّك التحليل وحده **1,718** سطرًا، ومحرّك الملاحظات **770**، وطبقة الترجمة **889**، وملف أنماط واحد **922**
- **تدفّق مفتاح API مكرّر** منسوخ بين شاشتين
- وبعض ملاحظات المحرّك **نفسه** كانت تحتاج دليلًا أدقّ

فسوّينا الصح: ضغطنا زرّ **«نسخ كـ Prompt»** في التقرير، وأعطينا تقرير الأداة الجنائي عن نفسها لمساعد ذكاء برمجي، وأصلحنا **بالضبط** ما سمّته — قسّمنا كل ملف عملاق إلى وحدات مركّزة تحت 400 سطر، واستخرجنا التدفّقات المشتركة، وشدّينا المحرّك. ثم **أعدنا الفحص بنفس الأداة**:

## كيف يشتغل؟

1. **الفحص** — يمشي على المجلّد ويتجاهل `node_modules` ومخرجات البناء و`.git` وملفات القفل والمولّدة.
2. **التحليل** — مقاييس الأسطر، الدوال الطويلة، التكرار، الاستيرادات غير المستخدمة، رسم الاعتمادات، وكشف أسرار مبدئي — كله في Rust ومحلّي.
3. **التقرير** — درجات وحكم مُقدَّر وملاحظات مرتّبة بأدلّتها وخطط إصلاحها.
4. **(اختياري) حلّل بالذكاء** — خلاصة مضغوطة (أرقام ومسارات، **لا كود**) تُرسل إلى DeepSeek لتقرير مكتوب.

```bash
cd code-crime-scene

```

## إعداد الذكاء (DeepSeek)

الذكاء **اختياري** — التقرير المحلي يشتغل كامل بدون مفتاح. لفتح "تقرير المحقّق" المكتوب:

1. أنشئ مفتاحًا من **[platform.deepseek.com](https://platform.deepseek.com/) ← API Keys** (يبدأ بـ `sk-…`).
2. افتح التطبيق ← **الإعدادات ← ذكاء DeepSeek** (أو الصقه في شاشة البداية).
3. الصق المفتاح واضغط **تحقّق واحفظ**. التطبيق يرسل طلبًا حقيقيًا صغيرًا لـ DeepSeek ليتأكّد أن المفتاح يعمل، ثم يحفظه في **خزنة النظام**.
4. افتح أي تقرير ← انزل إلى **تقرير المحقّق** ← اضغط **حلّل بالذكاء**.

<p align="center">
  <img src="assets/screenshots/ai-contacting.png" alt="لوحة تقرير المحقّق الاختيارية — لحظة الاتصال بـDeepSeek" width="88%">
</p>

**ضمانات**
- زرّ **«حلّل بالذكاء»** مخفي حتى تحفظ مفتاحًا، والـ backend يرفض التشغيل بدون مفتاح. لا مفتاح ⇐ لا استدعاء للذكاء، أبدًا.
- **كودك المصدري ما يُرسَل أبدًا**. تطلع أرقام مجمّعة ومسارات فقط، وفقط عند تلك الضغطة.
- المفتاح يُقرأ **في Rust فقط** عند استدعاء DeepSeek، وما يلمس طبقة الويب.

> حاليًا يدعم التطبيق **DeepSeek فقط**. إضافة موزّع آخر تعديل صغير ومحدّد — انظر تحت.

## إضافة موزّع/خدمة ذكاء أخرى (مفتوح المصدر 💚)

كل تكامل الذكاء في **ملف واحد**: [`src-tauri/src/ai.rs`](src-tauri/src/ai.rs)، فيه دالتان فقط:

```rust
pub async fn verify_key(key: &str) -> Result<(), String>;                 // التحقّق من المفتاح
pub async fn analyze(key: &str, model: &str, summary: &str, lang: &str)   // الحصول على التقرير
    -> Result<String, String>;
```

لإضافة **OpenAI أو Anthropic أو Gemini أو Mistral أو Ollama أو نموذج محلّي** أو أي نقطة متوافقة مع OpenAI:

1. وجّه الطلب إلى رابط `…/chat/completions` الخاص بموزّعك (أو الـ SDK).
2. أبقِ نفس **برومبت النظام** (`SYSTEM_PROMPT` / `SYSTEM_PROMPT_AR`) ليبقى شكل التقرير ثابتًا.
3. حوّل الردّ إلى نصّ Markdown.

بما أن جسم الطلب يتبع مخطّط OpenAI (`model`, `messages`, `temperature`, `max_tokens`)، فمعظم الموزّعين مجرّد **تبديل رابط + ترويسة مصادقة**. الواجهة وتخزين المفتاح واللغة وعرض التقرير تبقى كما هي. الـ PRs اللي تضيف موزّعين (ويا حبّذا خلف مبدّل `provider` بسيط في الإعدادات) مرحّب بها جدًا.

## الخصوصية والأمان

- **لا يطلع أي كود مصدري من جهازك.** استدعاء الذكاء الاختياري يرسل أرقامًا مجمّعة ومسارات فقط.
- **لا backend، لا تسجيل دخول، لا تحليلات.** التطبيق ما يعمل أي اتصال شبكي إلا لمّا تضغط «حلّل بالذكاء».
- **المفتاح في خزنة النظام** (Windows Credential Manager) — لا يُكتب على القرص كنصّ، ولا يُكشف للواجهة.
- **التقارير محلية** في SQLite داخل مجلّد بيانات التطبيق.
- كشف الأسرار محاولة ساكنة مبدئية — وهو **ليس** تدقيقًا أمنيًا.
